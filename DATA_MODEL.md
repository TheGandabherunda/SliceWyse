# Data Model & Math Specification — SliceWyse

## 1. Domain Entities

### `Money` (Value Object)

- `amountCents: number` (integer representation of minor currency units).
- `currency: string` (ISO code e.g. USD, EUR, JPY).
- Method `splitEqually(count)`: Distributes remainder cents to initial members deterministically to guarantee that $\sum \text{Splits} = \text{TotalAmount}$.

### `Group` (Aggregate Root)

- `id: string`
- `name: string`
- `currency: string`
- `members: Member[]`

### `Expense` (Entity)

- `id: string`
- `groupId: string`
- `title: string`
- `amount: Money`
- `paidBy: PayerShare[]`
- `splits: SplitShare[]`
- `splitType: 'EQUAL' | 'EXACT' | 'PERCENTAGE'`
- `version: number`
- `previousVersionId?: string`
- `isDeleted: boolean`

### `Settlement` (Entity)

- `id: string`
- `groupId: string`
- `payer: string` (pubkey)
- `payee: string` (pubkey)
- `amount: Money`

## 2. Debt Simplification Algorithm

Net balance for user $u$:
$$\text{NetBalance}(u) = \sum \text{PaidBy}(u) - \sum \text{OwedBy}(u) + \sum \text{SettlementReceived}(u) - \sum \text{SettlementPaid}(u)$$

Key Invariant:
$$\sum_{u \in \text{Members}} \text{NetBalance}(u) = 0$$

Direct transfers generated by `DebtSimplifier.simplifyDebts()` reduce the total transaction count from $O(N^2)$ to at most $O(N)$ transfers while preserving every member's exact net balance position.
